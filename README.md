# verify-assumptions
The repository was assembled gradually while experimenting with transaction monitoring, wallet interactions, and local testing environments. Many of the files solve very small problems and were originally created to verify assumptions during development.

Some examples examine signalling mechanisms between independent components. These experiments focus on how information travels through different stages of execution and how small state changes can be recorded.

A few utilities collect metrics related to trading volume in order to compare activity between separate runs. The intention is not market analysis but simply maintaining useful records during testing.

Several experiments also explore layer2 environments and the practical differences they introduce when transactions are prepared and verified. These examples were kept intentionally lightweight so they can be modified without large dependencies.

Because the repository evolved over time, individual scripts often follow different styles and structures. This variety reflects the practical nature of the project and the independent purpose of each file.
